Enterprise Information Systems Auditing

       

Automating continuous compliance, mapping risk frameworks, and validating AI security via Cloud Security Posture Management (CSPM) across a cloud infrastructure.

Lab 1: Audit Trail Non-Repudiation

1. Control Implementation: Secure Audit Storage (Amazon S3)

Configured an Amazon S3 bucket to serve as the centralized, immutable storage repository for environment-wide audit logs.

  • Bucket Designation: audit-trail-logs-rfs-2026
  • Access Control: Enforced "Block all public access" to mitigate risk of unauthorized external data exposure.
  • Data Encryption: Enabled Server-Side Encryption with Amazon S3 managed keys (SSE-S3) to ensure data confidentiality at rest.

2. Control Implementation: Continuous Audit Logging (AWS CloudTrail)

Deployed AWS CloudTrail to continuously record management events, API calls, and console actions across the AWS infrastructure, ensuring absolute non-repudiation of administrative activity.

  • Trail Designation: audit-trail-rfs-2026
  • Log File Validation: Enabled cryptographic hashing (SHA-256) to establish mathematical proof of log integrity and detect unauthorized tampering.
  • Audit Scope: Configured to capture 100% of Read and Write Management Events.

3. Audit Verification & Detective Control Validation

Executed verification procedures to confirm the implemented detective controls functioned precisely as designed.

  • Storage Verification: Confirmed the successful delivery and structural integrity of compressed .json.gz log files within the S3 architecture.
  • Identity Tracking: Successfully utilized CloudTrail Event History to trace administrative infrastructure changes directly back to the acting identity (capturing a high-risk root account login), validating the system's ability to enforce strict accountability.
