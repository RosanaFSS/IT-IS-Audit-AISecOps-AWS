Audit Trail Non-Repudiation

Create a Secure Audit Storage (AWS S3)

  1. Sign in to the AWS Management Console.
  2. In the top search bar, type S3 and select it from the results.
  3. Click the orange Create bucket button.
    - Bucket type: Leave as General purpose.
    - Bucket name: Type audit-trail-logs-XXX-XXXX.
    - Select an AWS Region.
    - Scroll down to Block Public Access settings for this bucket. Ensure the box for Block all public access is CHECKED (this is a critical security control).
    - Scroll to the very bottom and click Create bucket.

Configure AWS CloudTrail

  1. In the top search bar of the AWS console, type CloudTrail and select it from the results.
  2. On the right side dashboard, click the orange Create trail button.
  3. Type audit-trail-logsXXXXXXXXX.
  4. Scroll down to the Storage location section.
  5. Select the S3 bucket just created.
  6. Click the dropdown or browse button and select the bucket you just created (audit-trail-logs-XXX-XXXX).
  7. Log file SSE-KMS encryption: UNCHECK this box. (Creating a new KMS key incurs a monthly fee. The SSE-S3 encryption you enabled on the bucket earlier is free and sufficient).
  8. Log file validation: Leave this CHECKED. (This is a core IT auditing concept; it creates a cryptographic hash to prove log integrity and non-repudiation).
  9. Click Next at the bottom right.
  10. On the Choose log events screen, ensure Management events is checked. Leave Data events and Insight events unchecked to avoid extra costs.
  11. Scroll down to the Management events section and ensure both Read and Write are selected.
  12. Click Next, scroll to the bottom of the review page, and click Create trail.

Verify Audit Log Delivery

  1. Navigate back to the S3 service in the AWS Console.
  2. Click on your bucket name (audit-trail-logs-rfs-2026).
  3. Click through the generated folder structure: AWSLogs/ -> [Your Account ID] -> CloudTrail -> us-east-1 -> 2026 -> 07 -> 20.
  4. You will see compressed .json.gz log files. This verifies CloudTrail is actively recording administrative actions and securely storing them.

The presence of those .json.gz files in the folder is the exact proof required to demonstrate that your CloudTrail setup is successfully capturing and delivering audit logs.

Teardown (Zero Cost Enforcement)

To ensure no future charges accrue, delete the resources created.

  1. Go to the CloudTrail dashboard, click on audit-trail-rfs-2026, and click Delete. Confirm the deletion.
  2. Go to the S3 dashboard, select the radio button next to audit-trail-logs-rfs-2026, and click Empty. Type permanently delete to confirm.
  3. Once empty, select the bucket again and click Delete. Type the bucket name to confirm.
